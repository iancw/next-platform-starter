## ADDED Requirements

### Requirement: Sitemap lists all public recipe URLs
The app SHALL expose a `/sitemap.xml` generated by `app/sitemap.js` that includes one entry per public recipe, using the recipe's UUID (preferred) or slug as the URL path segment.

#### Scenario: Sitemap includes recipe URLs
- **WHEN** a crawler fetches `/sitemap.xml`
- **THEN** the response contains one `<url>` entry per recipe, each with a `<loc>` of the form `https://<domain>/recipes/<uuid-or-slug>`

#### Scenario: Sitemap uses UUID over slug
- **WHEN** a recipe has both a UUID and a slug
- **THEN** the sitemap entry uses the UUID (matching the canonical URL pattern)

#### Scenario: Sitemap returns valid XML
- **WHEN** `/sitemap.xml` is fetched
- **THEN** the response has `Content-Type: application/xml` and conforms to the sitemap protocol schema

### Requirement: Sitemap includes static pages
The sitemap SHALL also include entries for the static public pages: `/`, `/about`, `/how-to`.

#### Scenario: Static pages appear in sitemap
- **WHEN** `/sitemap.xml` is fetched
- **THEN** entries for `/`, `/about`, and `/how-to` are present

### Requirement: Private pages are excluded from the sitemap
Authenticated-only routes (`/my-samples`, `/profile`, `/user`, `/upload`, `/login`) SHALL NOT appear in the sitemap.

#### Scenario: Private routes absent from sitemap
- **WHEN** `/sitemap.xml` is fetched
- **THEN** none of `/my-samples`, `/profile`, `/user`, `/upload`, `/login` appear as `<loc>` values

### Requirement: Sitemap query is lightweight
The DB query used to build the sitemap SHALL select only the columns required for URL construction (`uuid`, `slug`) and SHALL NOT load recipe settings, images, or other large fields.

#### Scenario: Sitemap query selects minimal columns
- **WHEN** `app/sitemap.js` executes
- **THEN** it issues a SELECT limited to `uuid` and `slug` (and optionally `updatedAt`) from the recipes table
